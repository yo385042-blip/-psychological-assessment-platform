## ✅ 解决方案 2：使用代理（如果你有代理）

### 如果你有代理软件（如 Clash、V2Ray 等）：

#### 步骤 1：找到代理端口

- **通常是：** `127.0.0.1:7890` 或 `127.0.0.1:1080`
- **检查你的代理软件设置**

#### 步骤 2：配置 Git 使用代理

**在 PowerShell 中运行：**

```powershell
# 设置 HTTP 代理（替换端口为你的实际端口）
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy https://127.0.0.1:7890
```

#### 步骤 3：尝试推送

```powershell
git push -u origin main
```

#### 步骤 4：推送成功后，取消代理（可选）

```powershell
git config --global --unset http.proxy
git config --global --unset https.proxy
```

## ✅ 解决方案 3：使用 SSH 代替 HTTPS（推荐）

### 如果你有 SSH 密钥或愿意创建：

#### 步骤 1：检查是否有 SSH 密钥

**在 PowerShell 中运行：**

```powershell
ls ~/.ssh
```

**如果看到 `id_rsa` 或 `id_ed25519` 文件，说明已有密钥。**

#### 步骤 2A：如果没有密钥，创建新的

```powershell
ssh-keygen -t ed25519 -C "your_email@example.com"
```

- 按 Enter 使用默认路径
- 可以设置密码或直接按 Enter

#### 步骤 2B：查看公钥

```powershell
cat ~/.ssh/id_ed25519.pub
```

**复制输出的公钥内容。**

#### 步骤 3：添加 SSH 密钥到 GitHub

1. **访问：** https://github.com/settings/keys
2. **点击 "New SSH key"**
3. **填写：**
   - Title: `My Computer`
   - Key: 粘贴刚才复制的公钥
4. **点击 "Add SSH key"**

#### 步骤 4：更改远程仓库 URL 为 SSH

```powershell
git remote set-url origin git@github.com:yo385042-blip/mindcube.git
```

#### 步骤 5：尝试推送

```powershell
git push -u origin main
```
