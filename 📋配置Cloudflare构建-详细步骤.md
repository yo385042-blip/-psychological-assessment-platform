# 📋 配置 Cloudflare Pages 构建 - 详细步骤

## ✅ 我看到你的页面了！

你的 Cloudflare Pages 已经连接到 GitHub 仓库：`yo385042-blip/mindcube`

现在需要填写构建配置。

---

## 🎯 步骤 1：点击编辑按钮

1. **找到 "构建配置 (Build Configuration)" 卡片**
2. **点击卡片右上角的 ✏️ 编辑图标**

---

## 🎯 步骤 2：填写构建配置

点击编辑后，你会看到输入框，请按以下填写：

### 📝 填写内容：

| 字段名称 | 填写内容 | 说明 |
|---------|---------|------|
| **构建命令** (Build command) | `npm run build` | 构建项目的命令 |
| **构建输出** (Build output) | `dist` | 构建后文件存放的目录 |
| **根目录** (Root directory) | **留空** | 项目根目录，留空即可 |

### 📸 具体填写：

```
构建命令 (Build command):    npm run build
构建输出 (Build output):      dist
根目录 (Root directory):      (留空，什么都不填)
```

---

## 🎯 步骤 3：保存配置

1. **填写完成后**
2. **点击页面底部的 "保存" (Save) 按钮**
   - 可能是蓝色的按钮，写着 "保存" 或 "Save"
   - 可能在页面底部，也可能在配置卡片内部

---

## ✅ 完成后的效果

填写完成后，你的构建配置应该显示：

```
构建命令: npm run build
构建输出: dist
根目录: (空)
```

---

## 🔄 步骤 4：等网络恢复后推送代码

配置完成后，当网络恢复时：

1. **在 PowerShell 中运行：**
   ```powershell
   git push -u origin main
   ```

2. **Cloudflare Pages 会自动：**
   - ✅ 检测到新的提交
   - ✅ 运行 `npm run build`
   - ✅ 部署 `dist` 目录
   - ✅ 应用 `_redirects` 文件
   - ✅ 应用 `functions/_middleware.js`

---

- ✅ **之后每次推送代码都会自动部署**
- ✅ **代码已经在本地提交了，不会丢失**
