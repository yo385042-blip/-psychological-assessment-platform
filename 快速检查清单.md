# ✅ 快速检查清单

使用这个清单快速检查你的 Cloudflare Pages 部署状态。

---

## 📋 检查项

### 1. 文件结构检查

在项目根目录检查以下文件是否存在：

- [ ] `functions/_middleware.js` 存在
- [ ] `public/_redirects` 存在
- [ ] `dist/index.html` 存在（构建后）
- [ ] `dist/assets/` 目录包含 JS/CSS 文件（构建后）

### 2. Cloudflare Pages 构建状态

在 Cloudflare 控制台检查：

- [ ] 最新部署状态：✅ 成功 / ❌ 失败
- [ ] 构建日志：无错误信息
- [ ] 构建输出：`dist` 目录包含所有文件

**如何查看：**
1. 登录 https://dash.cloudflare.com/
2. Workers & Pages → 你的项目
3. Deployments 标签 → 查看最新部署

### 3. Functions 中间件状态

在 Cloudflare Pages 项目检查：

- [ ] Functions 页面显示 `_middleware.js`
- [ ] 状态为：Active 或 Deployed
- [ ] 无错误信息

**如何查看：**
1. 项目页面 → Functions 标签
2. 查看 `_middleware.js` 状态

### 4. 浏览器控制台检查

打开网站并检查：

- [ ] 无红色错误信息
- [ ] 无关键警告信息
- [ ] 页面正常加载

**如何检查：**
1. 按 `F12` 打开开发者工具
2. 切换到 Console 标签
3. 刷新页面，查看错误

### 5. 网络请求检查

在浏览器开发者工具中检查：

- [ ] 所有资源请求成功（状态码 200）
- [ ] 无 404 错误
- [ ] 无 500 错误
- [ ] 无 ERR_CONNECTION_CLOSED 错误

**如何检查：**
1. 开发者工具 → Network 标签
2. 刷新页面
3. 查看请求列表，确认无失败请求

### 6. 路由测试

测试不同路由是否正常工作：

- [ ] 首页 `/` 可以访问
- [ ] 登录页 `/login` 可以访问
- [ ] 仪表盘 `/dashboard` 可以访问（需要登录）
- [ ] 其他路由可以正常访问

---

## 🚨 发现问题时的操作

### 如果构建失败

1. **查看构建日志**
   - 复制错误信息
   - 截图构建日志

2. **检查构建配置**
   - 构建命令：`npm run build`
   - 输出目录：`dist`
   - Node.js 版本：18+

### 如果 Functions 未部署

1. **检查文件位置**
   - `functions/_middleware.js` 必须在项目根目录
   - 不在 `dist` 目录中

2. **检查 Git 提交**
   - 如果使用 Git，确认 `functions/` 目录已提交
   - 使用 `git status` 检查

### 如果浏览器有错误

1. **记录错误信息**
   - 截图控制台错误
   - 复制错误文本

2. **检查网络请求**
   - 截图 Network 标签
   - 记录失败的请求 URL
