### 方案 2：使用代理（推荐，如果有代理）

#### 如果你有代理软件（Clash、V2Ray、SSR 等）：

#### 步骤 1：找到代理端口

**常见的代理端口：**
- `127.0.0.1:7890` (Clash 默认)
- `127.0.0.1:1080` (V2Ray/SSR 默认)
- 查看你的代理软件设置

#### 步骤 2：配置 Git 使用代理

```powershell
# 设置 HTTP 代理（替换端口为你的实际端口）
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy https://127.0.0.1:7890
```

#### 步骤 3：推送

```powershell
git push -u origin main --force
```

#### 步骤 4：推送成功后，取消代理（可选）

```powershell
git config --global --unset http.proxy
git config --global --unset https.proxy
```

### 方案 3：使用 SSH 代替 HTTPS（最稳定）

#### SSH 通常比 HTTPS 更稳定，不易断线。

#### 步骤 1：检查是否有 SSH 密钥

```powershell
ls ~/.ssh
```

**如果看到 `id_rsa` 或 `id_ed25519`，说明已有密钥。**

#### 步骤 2：如果没有密钥，创建新的

```powershell
ssh-keygen -t ed25519 -C "your_email@example.com"
```

- 按 Enter 使用默认路径
- 可以设置密码或直接按 Enter

#### 步骤 3：查看并复制公钥

```powershell
cat ~/.ssh/id_ed25519.pub
```

**复制输出的完整内容（从 `ssh-ed25519` 开始到邮箱结束）。**

#### 步骤 4：添加 SSH 密钥到 GitHub

1. **访问：** https://github.com/settings/keys
2. **点击 "New SSH key"**
3. **填写：**
   - Title: `My Computer` 或任何名字
   - Key: 粘贴刚才复制的公钥
4. **点击 "Add SSH key"**

#### 步骤 5：更改远程仓库 URL 为 SSH

```powershell
git remote set-url origin git@github.com:yo385042-blip/mindcube.git
```

#### 步骤 6：测试 SSH 连接

```powershell
ssh -T git@github.com
```

**应该看到：** `Hi yo385042-blip! You've successfully authenticated...`

#### 步骤 7：推送

```powershell
git push -u origin main --force
```
