# 🌐 网络连接问题 - 解决方案

## ❌ 当前问题

- ✅ 远程仓库 URL 已修复为 `mindcube.git`
- ❌ 推送时遇到网络连接错误：
  - `Failed to connect to github.com port 443`
  - `Recv failure: Connection was reset`

**这是网络连接问题，不是代码问题！**

---

## ✅ 解决方案（多个选择）

### 方案 1：配置 Git 使用代理（如果你有代理）

#### 步骤 1：配置 HTTP 代理

**在 PowerShell 中运行：**

```powershell
git config --global http.proxy http://你的代理地址:端口
git config --global https.proxy https://你的代理地址:端口
```

**示例（如果是本地代理）：**
```powershell
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy https://127.0.0.1:7890
```

#### 步骤 2：重新推送

```powershell
git push -u origin main
```

#### 步骤 3：如果不需要代理了，取消代理

```powershell
git config --global --unset http.proxy
git config --global --unset https.proxy
```

---

### 方案 2：使用 SSH 代替 HTTPS（推荐，如果 SSH 可用）

#### 步骤 1：生成 SSH 密钥（如果还没有）

```powershell
ssh-keygen -t ed25519 -C "your_email@example.com"
```

**按 Enter 使用默认路径，可以设置密码或直接按 Enter。**

#### 步骤 2：添加 SSH 密钥到 GitHub

1. **查看公钥：**
   ```powershell
   cat ~/.ssh/id_ed25519.pub
   ```

2. **复制公钥内容**

3. **添加到 GitHub：**
   - 访问 https://github.com/settings/keys
   - 点击 "New SSH key"
   - 粘贴公钥
   - 保存

#### 步骤 3：更改远程仓库 URL 为 SSH

```powershell
git remote set-url origin git@github.com:yo385042-blip/mindcube.git
```

#### 步骤 4：推送

```powershell
git push -u origin main
```

---

### 方案 3：稍后重试（网络可能暂时不稳定）

1. **等待几分钟**
2. **重新尝试推送：**
   ```powershell
   git push -u origin main
   ```

---

### 方案 4：使用 GitHub Desktop（图形界面）

1. **下载 GitHub Desktop**
   - https://desktop.github.com/

2. **登录并打开仓库**

3. **点击 Push origin**

---

### 方案 5：先配置 Cloudflare Pages，稍后推送代码

如果网络一直有问题，可以：

1. **先配置 Cloudflare Pages 构建**（即使代码还没推送）
2. **等网络稳定后再推送代码**
3. **Cloudflare Pages 会自动检测并部署**

---

## 🎯 推荐操作顺序

### 立即尝试（方案 3）：

1. **等待 2-3 分钟**
2. **再次尝试推送：**
   ```powershell
   git push -u origin main
   ```

### 如果还是不行（方案 1 或 2）：

- **如果你有代理** → 使用方案 1
- **如果你想用 SSH** → 使用方案 2

### 如果都不行（方案 5）：

- **先配置 Cloudflare Pages**
- **稍后推送代码**
