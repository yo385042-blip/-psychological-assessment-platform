# 🚀 立即修复：配置重定向规则

## ❌ 问题

网站无法打开，需要配置路由重定向。

---

## ✅ 解决方案：在 Dashboard 配置重定向规则

这是最快的修复方法！

---

## 📋 详细操作步骤

### 步骤 1：进入项目

1. **登录 Cloudflare Dashboard**
   - https://dash.cloudflare.com/

2. **进入 Pages 项目**
   - Workers & Pages → 你的项目（psytest.store）

### 步骤 2：找到重定向配置

**方法 A：通过标签页**

1. **在项目页面顶部，查看标签：**
   - 资产已上传
   - 函数
   - **重定向** ← **点击这个！**
   - 标头
   - 实时日志

2. **点击 "重定向" (Redirects) 标签**

**方法 B：通过设置**

如果看不到 "重定向" 标签：

1. **点击 "设置" (Settings) 标签**
2. **在左侧菜单或页面中查找：**
   - **"Redirects"** 或 **"重定向"**
   - **"Routes"** 或 **"路由"**

### 步骤 3：添加路由规则

1. **点击 "添加规则" 或 "Create rule" 按钮**

2. **填写以下信息：**

   ```
   规则名称（可选）：SPA Routing
   匹配模式：/*
   重写为：/index.html
   状态码：200
   ```

   **或者如果界面显示字段名称不同：**

   - **Pattern（模式）：** `/*`
   - **Rewrite to（重写为）：** `/index.html`
   - **Status code（状态码）：** `200`
   - **Preserve query string（保留查询字符串）：** 可选，建议勾选

3. **点击 "保存" 或 "Save" 或 "Create"**

### 步骤 4：测试网站

1. **等待几秒钟**（配置通常立即生效）

2. **访问你的网站 URL**
   - 在部署详情页面查看 URL
   - 通常是 `https://psytest.store.pages.dev` 或类似

3. **测试功能：**
   - ✅ 首页是否能打开？
   - ✅ 路由（如 `/dashboard`、`/login`）是否能访问？
