# 📝 填写 Token 表单 - 详细说明

## ✅ 你现在在 GitHub Token 创建页面！

按照以下步骤填写表单：

---

## 🎯 步骤 1：填写 "笔记" (Note)

**在 "笔记" 输入框中填写：**

```
Cloudflare Pages Deploy
```

或者填写任何你喜欢的描述，比如：
- `Git Push Token`
- `Deployment Token`
- `mindcube项目部署`

---

## 🎯 步骤 2：选择 "到期日" (Expiration)

**点击下拉菜单，选择：**

- ✅ **推荐：** `90天`（如果列表中有）
- ✅ **或者：** `30天`（默认，也可以）
- ✅ **如果需要：** `No expiration`（无过期，但不太安全）

目前显示的是 `30天 (2025年12月28日)`，这个也可以使用。

---

## 🎯 步骤 3：选择权限范围（最重要！）

### 勾选 "仓库" (Repository)

**找到 "选择范围" (Select scopes) 部分，找到：**

```
☐ 仓库
   对私有仓库的完全控制权
```

**勾选这个复选框：** ✅

⚠️ **重要：**
- 勾选 "仓库" 会自动勾选所有相关的子权限
- 这是推送代码到 GitHub 必需的权限
- 你不需要手动勾选子选项，勾选主选项即可

---

## ✅ 步骤 4：生成 Token

1. **滚动到页面最底部**
2. **找到绿色的按钮：** "生成令牌" 或 "Generate token"
3. **点击按钮**

---

## 📋 步骤 5：复制 Token

1. **Token 会立即显示在页面上**
2. **Token 是一长串字符，类似：**
   ```
   ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
3. **立即复制整个 Token**（点击输入框，全选，复制）

⚠️ **警告：**
- Token 只显示一次！
- 如果关闭或刷新页面，Token 就找不回来了
- 必须立即复制并保存

---

## 🎯 步骤 6：回到登录对话框

1. **回到 Git 登录对话框**（之前打开的窗口）
2. **在 "Personal access token" 输入框中粘贴 Token**
3. **点击 "Sign in" 按钮**

---

## 📋 完整填写清单

### 你需要填写/选择的内容：

- ✅ **笔记：** `Cloudflare Pages Deploy`
- ✅ **到期日：** `30天` 或 `90天`（保持默认也可以）
- ✅ **权限范围：** ✅ 勾选 **"仓库" (Repository)**

### 不需要填写的：

- ❌ 不需要填写其他权限范围
- ❌ 不需要勾选子选项（会自动勾选）

---

## 🎯 现在请执行

### 快速操作：

1. ✅ **填写笔记：** `Cloudflare Pages Deploy`
2. ✅ **到期日：** 保持 `30天` 或改为 `90天`
3. ✅ **勾选权限：** ✅ 勾选 **"仓库"** 复选框
4. ✅ **点击 "生成令牌" 按钮**
5. ✅ **复制 Token**
6. ✅ **回到对话框粘贴并登录**

**填写完成后告诉我！** 🚀
